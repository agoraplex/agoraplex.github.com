ACL
An access control list. From Wikipedia, “a list of permissions attached to an object.” See ACLs don’t for a discussion of the weaknesses of ACLs and comparisons with the capability-based security model.
authentication
Establishment of the identity of a principal.
authorization
Assignment of permissions, the right to act on a resource, to a principal.

TODO:
capability-based security
See Wikipedia
HTTP
The Hypertext Transport Protocol, defined in RFC 2616 (“HTTP/1.1”).
HTTP method
One of the request methods specified in section 5.1.1 of RFC 2616 (“HTTP/1.1”) or its extensions (e.g., RFC 5789 (“PATCH Method for HTTP”)).
key
Private data used to sign and verify a Capability.
object facade
A proxy around an object, exposing a subset of the object’s interface.
permission
The right to act on a resource, i.e., to apply verbs to it.
principal
An identity (e.g., user, group, role) resolvable by authentication.
resource
Anything identifiable by a URI. Within a Pyramid application, this may be an element in the resource tree, but our use of the term is general, and framework-agnostic.
scope
A collection of permissions authorized by a capability
signature
A cryptographic hash of the contents of a Capability.

URI
As defined in RFC 3986: “A Uniform Resource Identifier (URI) is a compact sequence of characters that identifies an abstract or physical resource.”
verb
In general, an action to be applied to a resource (e.g., a method on an object). In a web application, this is usually an HTTP method.
